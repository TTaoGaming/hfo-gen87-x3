# Sandbox Audits Folder

> **Purpose**: Compliance audits and structural reviews

## Structure

```
audits/
└── [TOPIC]_AUDIT_[YYYYMMDD].md
```

## Guidelines

### ✅ What Goes Here

- Documentation structure audits
- Architecture compliance reviews
- HIVE workflow audits
- Stub/theater detection reports
- Gap analysis documents

### Naming Convention

```
[TOPIC]_AUDIT_[YYYYMMDD].md

Examples:
- DOCUMENTATION_STRUCTURE_AUDIT_20251230.md
- ARCHITECTURE_COMPLIANCE_AUDIT_20251230.md
- STUB_DETECTION_AUDIT_20251230.md
```

### Required Sections

Every audit MUST include:
1. **Current state analysis** - What exists now
2. **Issues identified** - Problems found
3. **Proposed changes** - Recommendations
4. **Implementation checklist** - Actionable steps
5. **Success metrics** - How to measure improvement

---

*Audit documents are persistent - they track evolution over time*
